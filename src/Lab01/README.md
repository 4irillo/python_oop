# Ответы на вопросы практического занятия №1

## Вопрос 1: Что такое инкапсуляция и зачем она нужна?

**Инкапсуляция** — это один из принципов ООП, который заключается в сокрытии внутреннего состояния объекта и предоставлении доступа к нему только через определенные методы (интерфейс).

**Зачем нужна:**
* Защита данных от некорректного изменения
* Контроль доступа к внутреннему состоянию
* Возможность изменить внутреннюю реализацию без изменения интерфейса
* Предотвращение случайных ошибок

**В моем классе:** Приватное поле `__number_of_rates` инкапсулирует количество оценок — оно изменяется только через метод `rate()`.

## Вопрос 2: В чем разница между атрибутами класса и экземпляра?

**Атрибуты класса** — общие для всех экземпляров, определяются в теле класса.
**Атрибуты экземпляра** — уникальные для каждого объекта, создаются в `__init__`.

**Пример:**
```python
class Teacher:
    # Атрибут класса (общий для всех)
    institution = "University"
    
    def __init__(self, name):
        # Атрибут экземпляра (уникальный)
        self.name = name
```

## Вопрос 3: Как работают property и зачем они нужны?

**@property** — декоратор, который позволяет превратить метод в атрибут, доступный для чтения.

**Зачем нужны:**
* Контроль доступа к данным
* Валидация при установке значений
* Вычисляемые свойства
* Совместимость (можно заменить атрибут на property без изменения интерфейса)

**В моем классе:** Используются getter-методы (`get_name()`, `get_age()`, `get_rank()`) для доступа к элементам `personal_info`.

## Вопрос 4: Какие магические методы вы реализовали и зачем?

**Реализованные методы:**

1. **`__repr__`** — возвращает подробное строковое представление объекта для отладки:
   ```python
   Teacher(name='Alice', age=35, rank='Senior Lecturer', ...)
   ```

2. **`__eq__`** — определяет равенство объектов (по имени, возрасту и предмету):
   ```python
   teacher1 == teacher2  # True, если совпадают ключевые атрибуты
   ```

3. **`__hash__`** — позволяет использовать объекты в множествах и как ключи словаря.

## Вопрос 5: Как организована валидация в вашем проекте?

**Валидация вынесена в отдельный модуль `validation.py`:**

1. **Принцип единственной ответственности** — валидация отделена от бизнес-логики
2. **Переиспользование** — одни и те же функции используются в конструкторе и setter'ах
3. **Расширяемость** — легко добавить новые проверки
4. **Тестируемость** — валидацию можно тестировать отдельно

**Пример использования:**
```python
from validation import validate_teacher_name, ValidationError

def update_name(self, new_name: str) -> None:
    validate_teacher_name(new_name)  # Проверка вынесена
    self.personal_info[0] = new_name
```
